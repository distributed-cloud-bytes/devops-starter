# Kikplate badges

This starter is published on [Kikplate](https://kikplate.dev/plates/event-driven-devops-starter). Badge shields in the root [README](../README.md) reflect the qualities the template is designed for.

## Target badges

| Badge | Slug | Why it applies |
|-------|------|----------------|
| Documented | `documented` | `docs/README.md` hub, architecture, configuration, operations, development |
| Docker Ready | `docker-ready` | `docker compose up`, Makefile targets, validated Compose CI |
| Well Tested | `well-tested` | Compose validation CI and k6 Schema Registry smoke under `loadtests/` |
| Cloud Native | `cloud-native` | Containerized Kafka-compatible broker, Schema Registry, Postgres |
| Beginner Friendly | `beginner-friendly` | Standalone quick start, empty topic catalog by default |
| Security Friendly | `security-friendly` | [SECURITY.md](../SECURITY.md), local-dev credential guidance |

## Official badge

The **Official** (`official`) badge is granted only by [Kikplate administrators](https://github.com/kikplate/kikplate). Plate owners can request it after the template is verified and documentation is complete.

To show badges on the Kikplate plate page (not only in this README), ask Kikplate maintainers to grant: `documented`, `docker-ready`, `well-tested`, and related community badges for plate slug `event-driven-devops-starter` under org `distributed-cloud-bytes`.
